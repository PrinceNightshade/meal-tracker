## Data Entry Format

All entries follow this pattern in api.js:

```javascript
{ name: 'Food Name', servingSize: X, servingUnit: 'unit', calories: Y, protein: P, carbs: C, fat: F, addedSugars: S, source: 'common', tags: 'tags' }
```

**Key points:**
- `addedSugars` must be numeric (0, 1, 5, 16, etc.)
- Use standard serving units from USDA (1 cup, 1 tbsp, 1 oz, etc.)
- Round to nearest 1g (no decimals needed for typical foods)
- Source stays as 'common'
- Tags help with search — add relevant descriptors

---

## Validation Before Committing

1. **No duplicates:** Check food name doesn't appear twice
2. **No overwrites:** Ensure items already with addedSugars aren't modified
3. **Realistic ranges:**
   - 0-2g: most proteins, vegetables, unsweetened items
   - 3-10g: sweetened breakfast, sauces, some snacks
   - 11-30g: desserts, sodas, sweet beverages
   - 30g+: candy, concentrated sweets
4. **Consistent formatting:** Match existing entry structure exactly
